Project description
paste into the form

Envelope is the treasury and payroll OS for companies that pay a distributed team in stablecoins. Customers pay invoices from any chain — or by card, UPI, or SEPA — and it all settles as USDC into one treasury with no single private key. Compensation lives encrypted before it touches a database; an on-chain program enforces the monthly cap and requires co-signers, so nobody reads a salary to approve a run. One approval pays every contractor in a single shielded batch: the public ledger shows one entry — no names, no amounts, no roster — and each recipient gets a one-time stealth claim link, greeted by their .sol handle.

The thing nobody else does: payroll on a public chain that doesn’t doxx your team. Deel, Rise, and Request all settle in plaintext — every salary, every raise, your whole burn rate, readable forever by competitors and recruiters. Envelope makes private the default, not a mode you remember to switch on. It runs on Solana, where the stablecoin volume already is and the privacy primitives shipped this cycle, and a 30-person batch costs cents in fees.

Built for the AI/SaaS founder paying ~30 contractors across 12 countries who can’t move payroll on-chain today without leaking strategy. The full loop runs against the chain — envelope.sol is registered, the envelope-policy program is deployed, and a real FHE compute gates every compensation policy.

How it works · the loop
three steps
  • IConnect your treasury. Bring your wallet and your contractor roster. Pay-ins land from any chain or a card and settle into one treasury secured by multi-party custody — no single key.
  • IISet the policy. Salary bands, a monthly cap, co-signers. Compensation is encrypted before it hits a database; an on-chain program enforces the cap and gates every release with a real FHE check.
  • IIIRun payroll. Approve once — every contractor is paid in a single shielded batch. The public ledger shows one entry; each recipient gets a one-time claim link tied to their .sol handle, and the orchestrating agent itself signs as payroll-agent.envelope.sol.
Track fit · what each protocol does in the loop
submission metadata
ProtocolJob in Envelope — load-bearing, not bolted on
KIRAPAYThe crypto pay-in rail. Customers pay an invoice from any chain in any token; it routes and settles to the treasury. Without it, Envelope can’t accept money in. lib/kirapay/*, app/checkout/[id].
Dodo PaymentsThe fiat pay-in rail — card, UPI, SEPA — plus Envelope’s own subscription billing. Fiat customers in, recurring revenue rail out. lib/dodo/*, app/api/dodo/webhook.
IkaThe treasury is an MPC dWallet — no single private key; spending caps and co-sign enforced on-chain. Without it, this is a hot wallet. lib/ika/*.
Encrypt (FHE)The compensation policy is encrypted — a real #[encrypt_fn] checks “is this amount within band?” over ciphertexts, so an approver never reads a salary. programs/envelope-policy/src/lib.rs.
CloakPayroll settles as one shielded batch — 30 payouts, one ledger entry, stealth claim links, scoped viewing keys for audit. Remove it and every salary is public forever. lib/cloak/*, app/api/payroll/run, app/claim/[id].
SNS.sol identity: contractors onboard by handle (with verified Records V2 — gh/tw/dc/email), claim pages greet them by name, and the orchestrator agent has a verifiable on-chain identity via the payroll-agent.envelope.sol subdomain. lib/sns/*, scripts/sns-bootstrap.ts.

Each protocol occupies a different structural axis of one product. Pull any out and the loop collapses — that’s the integration-depth answer for every track.

Under the hood & status
for judges who’ll run it

Stack: Next.js 14 (App Router, TS strict), @solana/web3.js v2, Helius RPC, Privy auth · Anchor program envelope-policy built on the encrypt-anchor crate · pnpm workspaces. One wrapper module per protocol under apps/web/lib/<protocol>/; all payroll mutations funnel through app/api/payroll/run/route.ts.

On Solana: Cloak shielded transfers run on mainnet; envelope-policy and the .sol identity layer are deployed on devnet (pnpm sns:bootstrap registers envelope.sol + the agent subdomain); envelope.sol resolves and the full payroll loop runs against the chain. (Encrypt’s FHE compute is pre-alpha — one real #[encrypt_fn] ships; we’re explicit in the README about the pre-alpha plaintext caveat.)

Run it: pnpm install && pnpm dev · pnpm sns:bootstrap for the .sol identities · README has the per-protocol setup + program IDs.
